课程资料处理链路

把课程资料转成可检索、可引用的学习上下文。

Brevyn 不把文件、课程和 AI 会话拆散处理。资料进入课程空间后，会围绕解析、索引、检索和任务会话形成一条连续工作流。

课程资料 PDF / PPTX / DOCX OCR 解析 RAG 检索 Embedding 本地向量索引 文件证据 Agent 会话
学期 按学期聚合课程、任务和近期活动，保留长期学习结构。
课程 每门课都有独立资料空间，reading、slides、rubric 和笔记不会混在一起。
任务 essay、exam、project 和 revision 以任务为单位承载文件、目标和会话记录。
资料 资料先解析成可检索证据，再进入 RAG 和 Agent 的上下文范围。
01 / 导入

课程资料归档

把 reading、rubric、lecture slides、draft 和截图归入对应课程或任务，建立清楚的资料边界。

课程 作业 Reading Rubric
02 / 解析

本地解析管线

将 PDF、PPTX、文档和作业要求拆成可检索片段；扫描件、截图和课表类材料可接入 OCR / Vision。

PDF 解析 PPTX 解析 OCR / Vision 索引队列
03 / 检索

RAG 证据检索

回答优先从当前课程和任务的已索引资料中召回证据，用于概念解释、论文论据和 rubric 条件核对。

语义检索 课程范围 引用来源 文件证据
04 / 向量

本地向量索引层

Embedding 和向量检索围绕本地工作区运行，索引按课程和任务隔离，资料更新后可以重新构建。

Embedding 向量检索 本地存储 可重建索引
05 / Agent

面向任务的 Agent

从 essay、exam、project 启动上下文会话，带上相关资料、任务目标和历史记录；本地文件操作保留确认流程。

Essay Exam Project 权限确认

功能界面

课程、资料和任务，在同一套桌面界面里协作。

Brevyn 的界面围绕课程空间设计：从创建课程、归档资料，到启动任务会话和追溯文件证据，都保留在同一个学习工作台里。

课程空间 资料归档 本地索引 RAG 上下文 任务会话
文件栏与会话上下文 会话文件、课程文件和学期资料在右侧 rail 中展开，适合检查当前 Agent 能看到哪些上下文。
会话文件 课程文件树 学期资料
来源范围与索引状态 来源面板区分学期资料和本学期课程资料，清楚显示当前范围、文件数量和待索引状态。
来源面板 范围控制 索引状态
课程创建 课程名称、代码、教师和资料会进入同一个课程空间，后续索引、检索和 Agent 会话都围绕这个空间展开。
课程空间 资料归档 本地索引 RAG 上下文
